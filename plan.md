# 分类重命名布局与批量导入 Plan

## 1. 需求文档

当前需要修复两个用户路径：

- 资源页管理模式中，分类重命名时“保存/取消”按钮不能溢出容器，必须在手机宽度内稳定展示。
- 从未归类材料导入到某个分类时，只保留批量选择、全选、取消选择和“导入已选”，不再在每行放单个导入按钮。

业务完成标准：用户可以在资源页安全重命名分类；用户可以在导入已有材料页勾选一个或多个材料，用同一个“导入已选”命令移动到目标分类，也可以取消选择。

## 2. 当前事实

- 资源页重命名区域仍使用 Vant 小按钮，虽然外层加了 grid，但实际运行中仍会溢出，说明组件内部宽度规则不适合这个窄行场景。
- 项目里其他稳定的小命令按钮使用 `view` 承载固定高度、padding、line-height 和明确宽度，不依赖 Vant 内部最小宽度。
- `pages/import-existing/import-existing` 当前同时有每行“导入”和底部“导入已选”，语义重复。
- 批量导入通过 `moveMaterialByMode(mode, materialId, targetLibraryId)` 完成，本地和云端接口一致。
- 导入已有材料页只允许从未归类材料导入到普通用户分类，不允许导入到未归类材料或公共资源。

## 3. 失败测试

完成前以下情况代表失败：

- 重命名时“保存/取消”按钮超出卡片或把文字挤压到不可读。
- 导入已有材料页不能选择多个材料。
- 没有全选或取消选择入口。
- 导入已有材料页仍出现每行“导入”和底部“导入已选”两个入口。
- 批量导入只改 UI，不调用真实移动接口。
- 批量导入失败后页面状态不恢复，或用户不知道失败。
- 选择一个材料后无法通过“导入已选”完成导入。

## 4. 目标

- 修复资源页重命名布局：移除重命名区域的 Vant 按钮，改为项目内稳定的 `view` 命令按钮，按钮宽度、gap、容器约束明确。
- 导入已有材料页增加选择模式：
  - 每行显示可点击复选框。
  - 点击行或复选框切换选择状态。
  - 顶部显示已选择数量。
  - 支持全选当前候选材料。
  - 支持取消选择。
  - 支持批量导入已选材料。
- 批量导入逐个调用 `moveMaterialByMode`，全部完成后同步云端缓存并回到目标分类。
- 删除每行“导入”按钮和对应单项导入页面方法，当前只保留“导入已选”一个提交入口。
- 更新测试和文档事实。

## 5. 不做范围

- 不做跨分类批量导入。
- 不做批量导入事务回滚。
- 不做部分失败重试队列。
- 不做搜索筛选候选材料。

## 6. 设计

资源页布局：

- `resource-edit` 继续作为重命名编辑容器。
- `rename-actions` 使用稳定 grid 布局，按钮容器 `min-width: 0`，按钮自适应两列。
- `rename-command` 采用和项目其他小命令按钮一致的 `view bindtap` 实现，明确高度、padding、line-height、文字居中和 `overflow-wrap`，不让 Vant 内部默认宽度参与计算。

批量导入：

- `CandidateMaterial` 只保留 `isSelected`。
- 页面 data 增加 `selectedMaterialIds`、`isBatchMoving`。
- `toggleCandidateSelection(id)` 切换选择。
- `selectAllCandidates()` 选择所有候选材料。
- `clearSelection()` 清空选择。
- `importSelectedMaterials()` 顺序调用 `moveMaterialByMode`。
- 导入中禁止重复触发，也禁止继续修改选择状态，底部提交按钮显示导入中。

错误处理：

- 未选择材料时点击批量导入，toast 提示。
- 批量失败时 toast 失败原因并重新加载候选列表。
- 云端批量导入完成后调用 `syncCloudData()`。
