# 分类删除、材料编辑、音频替换与播放控制 Plan

## 1. 需求文档

当前材料管理还不完整：分类只能新增不能删除，材料不能编辑，音频不能在本地导入或替换，播放控制缺少“从头开始”。这些都是材料库产品的基础能力，必须一次性补齐。

完成后用户可以：

- 删除空分类。
- 编辑材料标题和正文。
- 本地导入时可选音频。
- 已有材料可替换音频。
- 播放、暂停、继续、从头开始。

业务完成标准：

- 不保留空接口或只 toast 的假接口。
- 本地和云端路径都有真实接线。
- 分类非空时禁止删除，避免误删材料。
- 默认本地分类禁止删除，避免本地入口失效。
- 音频替换后材料只使用新主音频。
- 验证、测试、构建通过。

## 2. 当前事实

- 资源页已有新增分类，没有删除分类。
- 材料详情页显示原文，云端可上传/生成音频，本地只 toast “不上传云音频”。
- 导入页只导入文本，不支持选择音频。
- `saveListeningAudio` 云函数只新增音频，不替换旧音频记录。
- 本地仓储已有材料保存、分类新增、移动、排序、删除材料。
- 播放服务已有播放/暂停/继续，没有从头开始。
- 当前没有材料编辑接口。

## 3. 失败测试

完成前以下检查代表缺口：

- `pages/material-detail/material-detail.ts` 中本地上传音频只 toast。
- `pages/import/import.ts` 没有音频选择状态。
- `resources` 页面没有删除分类入口。
- `runtimeData` 没有删除分类、更新材料、替换音频接口。
- `audioPlayer` 没有 restart 能力。

## 4. 目标

- 增加删除分类接口：本地和云端均禁止删除非空分类，默认本地分类不可删。
- 增加材料编辑接口：标题、正文可编辑，不影响音频绑定。
- 增加音频替换接口：本地保存本机文件路径，云端上传云存储；替换时只保留当前主音频记录。
- 导入页支持可选音频，保存材料后立即绑定音频。
- 详情页支持编辑标题/正文、替换音频；本地和云端都可用。
- 播放服务支持从头开始，材料页展示播放/暂停/继续和从头开始。
- 补测试覆盖分类删除保护、材料编辑、音频替换、从头播放。

## 5. 不做范围

- 不做分类级联删除。
- 不做多音轨。
- 不做音频裁剪、录音和波形。
- 不做图片导入编辑。
- 不做真实云端部署验证。

## 6. 设计

### 6.1 数据接口

新增 API 类型：

- `UpdateMaterialRequest/Response`
- `DeleteLibraryRequest/Response`
- `ReplaceListeningAudioRequest/Response`

本地服务对应：

- `updateLocalMaterial`
- `deleteLocalLibrary`
- `replaceLocalMaterialAudio`

云端函数对应：

- `updateMaterial`
- `deleteLibrary`
- `replaceListeningAudio`

### 6.2 页面

资源页管理模式：

- 每个分类显示删除操作。
- 非空分类删除由服务拒绝并 toast 错误。

导入页：

- 增加“选择音频”入口。
- 音频可选；保存材料后如有音频则绑定。

详情页：

- 标题和正文可编辑。
- 保存修改。
- 替换音频。
- 云端保留生成音频入口。

材料页：

- 主按钮播放/暂停/继续。
- 新增“从头开始”操作。
