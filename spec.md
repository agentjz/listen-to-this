# 每日英语听写 Spec

`spec.md` 是“每日英语听写”的当前产品事实主干。它只记录已经明确并由当前工程承载的产品定位、业务边界、核心概念、用户路径和验收标准。

## 1. 产品定位

每日英语听写是一个微信小程序 + CloudBase 项目，用于把英文材料和可选音频组织成可听、可读、可导入、可删除、可管理的学习材料。

产品核心不是课程平台、语法老师、口语评分器或社交学习社区，而是围绕个人英文材料的听力和阅读工具。

## 2. 当前主线

- 用户可以使用本地接口，不依赖 CloudBase。
- 用户可以使用云端接口，数据进入 CloudBase。
- 本地接口和云端接口分离，不自动同步。
- 用户可以新增分类，并把材料加入指定分类。
- 用户可以删除空分类；默认本地分类和非空分类不能删除。
- 导入内容按完整材料保存。
- 导入材料时可以只导入文本，也可以同时选择音频。
- 材料有音频就播放。
- 材料没有音频就作为阅读材料。
- 材料页是主要操作界面，支持直接播放、暂停、继续、从头播放、查看原文和管理删除。
- 原文详情页支持编辑标题、编辑正文和替换音频。

## 3. 核心概念

### 3.1 资料库

资料库是材料来源的最高层组织。

当前资料库类型：

- 本地资料库：保存在当前设备的小程序缓存中。
- 云端资料库：保存在 CloudBase 中。
- 系统库/通用听力：后续由管理员维护的云端资料来源。

当前界面中，资料库也承担用户自定义分类职责。用户可以新增分类，把材料移动到其他分类，并在分类内长按拖拽调整材料顺序。

用户可以删除空的自定义分类。默认本地分类和包含材料的分类不能删除。

### 3.2 材料

材料是产品的基本学习单位，可以是一句话、一段话或一篇文章。

材料包含：

- 名称：列表和听力入口展示。
- 英文内容：用于原文展示和阅读。
- 图片附件：用于辅助查看材料，不参与文本转语音或播放。
- 音频数量：用于判断是否可以直接播放。
- 排序值：用于分类内长按拖拽排序和稳定展示。

### 3.3 听力音频

听力音频是材料可播放的音频文件。音频来源包括：

- 用户上传音频。
- 云端生成音频。

用户可以在导入材料时选择音频，也可以在原文详情页替换材料音频。替换音频后，材料使用新的当前音频记录。

上传音频支持：

- `mp3`
- `m4a`
- `wav`

当前不做整篇音频自动对齐，不做音频自动切句。

## 4. 用户路径

### 4.1 本地使用

1. 用户打开小程序。
2. 用户进入本地资源。
3. 用户看到本地资料库和材料。
4. 用户在材料页直接播放有音频的材料，或点材料标题查看原文。
5. 用户进入管理模式，可以导入或删除本地材料。
6. 用户可以编辑本地材料，并替换本地材料音频。

### 4.2 云端使用

1. 用户打开小程序。
2. 用户进入云端资源。
3. 小程序通过 CloudBase 读取云端资料库和材料。
4. 用户在材料页直接播放有音频的材料，或点材料标题查看原文。
5. 用户进入管理模式，可以导入或删除云端材料。
6. 用户可以编辑云端材料、上传替换音频，或通过已配置的 TTS 生成音频。

## 5. 当前功能范围

当前工程包含：

- 微信小程序 TypeScript 工程。
- Vant Weapp npm 依赖。
- 本地资源和本地导入。
- 云端资源和云端导入。
- 材料保存、读取和删除。
- 材料编辑和音频替换。
- 分类新增、删除空分类、材料移动分类和分类内长按拖拽排序。
- 首页、资源、材料、原文详情、导入、管理页面。
- 本地测试样例 `local-assets/dog-wolf-friendship/`。
- 自动测试、类型检查、乱码检查、坏命名检查和页面入口检查。

当前工程不包含：

- 已部署云环境。
- 已创建真实数据库集合。
- 已配置真实腾讯云 TTS 密钥。
- 已内置大批系统材料。
- 微信开发者工具中的 `miniprogram_npm/` 构建产物。

## 6. 不做范围

当前产品不做：

- 自动分句。
- 逐句播放。
- 听写输入和自动评分。
- 音频导出。
- 语法教学。
- 语法分析。
- 口语评分。
- 发音纠错。
- 作文批改。
- AI 老师聊天。
- 社交社区。
- 课程商城。
- 复杂考试题库。
- 多级分类体系。
- 材料市场。
- 自动联网查词。
- 图片 OCR。
- 图片内容转语音。
- 整篇音频自动对齐。
- 音频自动切句。

## 7. 数据边界

当前已确认的数据对象：

- `SourceLibrary`：资料库。
- `Material`：材料。
- `ListeningAudio`：听力音频。
- `DictionaryEntry`：词义记录。

本地数据保存在当前设备或微信开发者工具的小程序缓存中，当前缓存键为 `listen.localData`。云端数据保存在 CloudBase 中。本地和云端不自动同步。

`local-assets/` 是仓库级测试样例目录，不是小程序运行时写入目录。用户在小程序里本地导入的材料不会自动生成或修改 `local-assets/` 文件。

`local-assets/<folder-id>/` 中的 `<folder-id>` 是样例材料唯一标识，并用于派生默认标题。文件夹名使用小写字母、数字和连字符。样例材料文件使用固定通用命名：`text.txt`、`audio.mp3|m4a|wav`、`image.svg|png|jpg|jpeg`。

## 8. 验收标准

- 首页能进入本地资源、本地导入、云端资源、云端导入。
- 本地资源无需云环境即可显示样例材料。
- 材料页支持查看原文、直接播放、管理导入和删除。
- 材料页支持播放、暂停、继续和从头开始。
- 资源页支持新增分类、删除空分类，材料管理模式支持移动分类和长按拖拽排序。
- 详情页支持编辑材料标题、正文和替换音频。
- 导入页支持可选音频，并在保存材料后绑定到该材料。
- 没有音频的材料作为阅读材料存在，不阻塞导入和查看。
- `npm.cmd run verify` 通过。
- `npm.cmd run build` 能生成微信开发者工具运行所需 JS。
